# 0004: Mock fixtures via provider pattern

Status: Accepted

## Context

Local development sometimes happens without access to a running stetsom-api instance. The site still needs to render pages and exercise the admin panel end to end during that time, without diverging from the real API contract.

## Decision

Setting `USE_MOCK_DATA=1` in `.env.local` serves GET requests from local fixtures instead of a running API. Mutations return `{ _mock: true }` and do not persist.

- Fixtures live in `src/lib/mock/*.ts` and conform to the Orval-generated types in `@/api/stetsom/model`.
- `src/lib/api/providers/mock-provider.ts` consumes the fixtures and dispatches by URL-segment key.
- `src/lib/mock/data.json` is generated by `pnpm mock:dump` against a real API. It is never hand-edited.

The mock provider is a local, offline fallback only. Hosted `develop` deploys and PR previews always use the dedicated develop API, never mock data.

## Consequences

Fixture data has to stay contract-compatible with whatever stetsom-api currently returns, which means `pnpm mock:dump` needs to run again after any API contract change that affects mocked endpoints. Mock-only behavior must not be added, since the goal is an offline mirror of the real API, not a separate code path.

Full scope and dispatch rules: `.agents/rules/mock-api-guidelines.md`.
