# API Integration — Patterns & Rules

Types and React Query hooks are **auto-generated by Orval** from the stetsom-api OpenAPI spec.
- Models: `src/api/stetsom/model/` (barrel: `index.ts`)
- Hooks: `src/api/stetsom/endpoints/<tag>/<tag>.ts`

## Orval Client — Two Targets, Two Entry Points

Two Orval targets generate from the same spec. Both are auto-generated.

| Context | Import | Route |
|---|---|---|
| Client Component / React Query | hooks `useGetApi*` from `@/api/stetsom` (mutator `orvalClient`) | `/api/bff/*` (HttpOnly cookie injected server-side) |
| Server Component / Server Action | functions `getApi*` from `@/api/stetsom/server/<tag>/<tag>` (mutator `serverOrvalClient`) | Direct to stetsom-api |

- The `server/**` tree imports `next/headers` — import it ONLY from server code. It is never re-exported by the client barrel `@/api/stetsom`, or the client build fails (Turbopack poisoning).
- Never call `fetch()` directly for stetsom-api data — use the generated hooks (client) or the generated `server/**` functions (RSC)
- Hooks: `useGetApi*` for reads; `usePostApi*`, `usePatchApi*`, `useDeleteApi*` for mutations
- Include locale in React Query `queryKey` for any hook that returns locale-variant data

## Auth Pattern

1. `POST /api/auth/login` → `{ accessToken, refreshToken }` stored as `admin_token` HttpOnly cookie
2. Protected requests: cookie read server-side → `Authorization: Bearer <token>` forwarded automatically by BFF
3. `POST /api/auth/refresh` + `{ refreshToken }` → `{ accessToken }`

- `admin_token` is HttpOnly — never readable from JS; BFF handles forwarding
- Logout: `DELETE /api/auth/logout` (stateless; JWT expiry is the security boundary)
- **Public** (no auth): `/api/pages/*`, `/api/categories`, `/api/products` listing/detail, `/api/contact`, `/api/partner-locations`
- **Protected** (require Bearer): `/api/dashboard`, `/api/users`, `/api/banners`, `/api/library`, `/api/messages`, `/api/audit`, `/api/config`, `/api/upload/*`, `/api/products/admin*`, `/api/attributes`, `/api/templates`

## Error Contract

Errors throw `OrvalApiError` (from `@/api/stetsom/orval-client`). Use `toApiError(err)` to cast unknown throws.

```ts
err.status  // HTTP status number
err.code    // "UNAUTHORIZED" | "NOT_FOUND" | "CONFLICT" | "VALIDATION_ERROR" | ...
err.message // human-readable — display only, never branch on this
```

| Status | `code` |
|--------|--------|
| 401 | `UNAUTHORIZED` |
| 404 | `NOT_FOUND` |
| 409 | `CONFLICT` |
| 422 | `VALIDATION_ERROR` |
| 503 | `SERVICE_UNAVAILABLE` |

## Pagination

| Style | Response | Query params | Endpoints |
|---|---|---|---|
| Page-based | `{ items, total, page, pageSize, totalPages }` | `page` (1-indexed), `pageSize` (max 100) | Products catalog/CMS |
| Offset-based | `{ items, total }` | `limit` (max 200), `offset` | Users, messages, audit, library, banners |

Param types are auto-generated (e.g. `GetApiProductsParams`). Never mix styles on the same endpoint.

## Upload Flow (3 steps, no BFF stream)

```
1. POST /api/upload  → { uploadUrl, file_url, headers, ... }
2. PUT {uploadUrl}   (browser → S3 — send ONLY presign.headers, no Authorization)
3. POST /api/upload/complete  → { asset: LibraryAsset }
```

- `file_url` from step 1 = permanent public URL — store in resource fields
- `uploadUrl` expires (15 min) — never cache or reuse
- Always call step 3 — asset is not registered in the library until then

## User Roles

| Role | Access |
|------|--------|
| `SUPER_ADMIN` | Full, including user management |
| `ADMIN` | Content management, no user admin |
| `EDITOR` | Read + draft; no publish or destructive |

Server enforces roles; frontend uses them for UI display only.

## Site Page Payloads

`GET /api/pages/:slug` (home, about, catalog, support) returns a full page payload — one request for all page data. Do not split into multiple fetches. Returns `503` (not `404`) on missing content dependency — handle with fallback state, not 404 page.

## Audit Log

`AuditEntry.action`: `CREATE | UPDATE | DELETE | LOGIN | LOGOUT | PUBLISH`. `action_sentence` is pre-formatted — render directly. Filter by `entity` query param.

## Admin Route Auth Guard

Every admin route handler must start with:

```ts
const cookieStore = await cookies();
const token = cookieStore.get("admin_token")?.value;
if (!token) return unauthorizedResponse();
```

This is the **sole security boundary** for the *dedicated* route handlers (`/api/upload`, `/api/auth/*`) — no fallback layer. A handler that omits this guard is publicly accessible.

**Exception — the generic BFF passthrough (`/api/bff/[...path]`):** it does **not** validate the JWT itself. It forwards the HttpOnly `admin_token` as a Bearer header and relies on the upstream stetsom-api to enforce auth and roles. Anonymous browser requests simply arrive without a token and upstream returns `401`. Do not add a per-path guard here — the BFF is a transparent proxy by design; the boundary is upstream. The `proxy.ts` middleware (JWT check) guards admin **UI** routes as defense in depth.

## CMS vs Public — Context-Oriented BFF

| Context | i18n fields | Data |
|---|---|---|
| **Public** (`/api/products/*`, `/api/pages/*`, `/api/categories`) | Flat `string`, already locale-resolved by the API | Latest active file only |
| **CMS** (`/api/products/admin/*`, `/api/pages/:slug/cms`) | Raw `I18nString { pt, en?, es? }` | All file versions |

- Public components: use `PublicProductDetailPayload`, `PublicCategorySchema`. Never call `pickLocale` — the API already resolved it.
- CMS components: use `I18nString` types. Display all languages for editing.
- Include `?locale=` query param in public site data fetches. Add locale to React Query `queryKey` for locale-variant hooks.
