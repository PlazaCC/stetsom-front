# Security Policy

## Supported Versions

Stetsom Front is continuously deployed. Only the version in production from `main` is supported.

| Version | Supported |
| --- | --- |
| `main` (production) | Yes |
| Anything else (older commits, forks, PR previews) | No |

Security fixes land on `develop` and follow the release process in `CONTRIBUTING.md`.

## Reporting a Vulnerability

Report vulnerabilities through GitHub's private reporting feature:

1. Go to the **Security** tab of this repository.
2. Select **Report a vulnerability**.
3. Describe the affected URL or endpoint, reproduction steps, and potential impact.

The report creates a private draft advisory visible only to maintainers. Do not include vulnerability details in public issues.
